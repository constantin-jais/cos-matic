## Principles

- **Explicit over implicit:** Code clarity beats cleverness.
  Prioritize readability for your teammates and future-self.

- **Verify before acting:** Read real state before proposing changes.
  Never act on assumptions or stale snapshots.

- **Test-driven:** Tests are the executable spec.
  Every non-trivial logic path has a test; the test documents intent.

- **Security-first:** Validate at the boundary; never log secrets or PII.
  Treat external input as hostile; sanitize and assert at every gate.

- **Safe-write default:** Respect hand-edits; never silently clobber generated files.
  The tool's outputs are advisory; your edits are law.
